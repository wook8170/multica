# Multica Agent Guidelines (Project Specific)

본 지침은 이 프로젝트 내에서 작업을 수행하는 모든 에이전트가 반드시 준수해야 할 '데이터 일관성 및 파일 관리' 수칙입니다.

## 1. 파일 생성 및 관리 원칙 (File Management Rules)
- **로컬 워크스페이스 우선**: 모든 파일 생성, 수정 및 일시 저장 작업은 반드시 현재 작업 디렉토리(Current Working Directory) 내에서만 이루어져야 합니다.
- **임의 경로 금지**: `/tmp`, `~` 등 워크스페이스 외부의 전역 경로에 파일을 생성하는 행위를 엄격히 금지합니다.
- **일관된 위치 사용**: 작업 유형에 따라 파일 위치를 바꾸지 말고, 항상 프로젝트의 표준 디렉토리 구조를 따르십시오.

## 2. 외부 서비스 직접 접근 금지 (Platform Abstraction)
- **중앙 통제 준수**: GitHub CLI(`gh`), Minio Client(`mc`), AWS CLI 등을 사용하여 외부 저장소나 서비스에 직접 파일을 전송하거나 생성하지 마십시오.
- **공식 API 권장**: 모든 서비스 상호작용(이슈 업데이트, 댓글 작성, 파일 첨부)은 반드시 `multica` CLI를 통해서만 수행해야 합니다.

## 3. Git 및 소스 관리 (Git Version Control)
- **로컬 Git 활용**: 소스 코드 수정 사항은 항상 로컬 Git을 통해 관리(add, commit)하십시오.
- **워크트리 사용**: `multica repo checkout`을 통해 할당받은 브랜치와 워크트리 내에서만 작업하십시오.
- **원격 푸시**: 작업을 마친 후에는 로컬 커밋을 원격으로 푸시하되, 외부 도구를 이용해 깃허브 웹상에서 직접 파일을 생성/편집하는 등 로컬 워크플로우를 우회하지 마십시오.

## 4. 데이터 공유 및 저장 (Data Sharing)
- **직접 업로드 금지**: 파일을 S3, Minio, Google Drive 등 외부 저장소에 직접 업로드하지 마십시오.
- **플랫폼 내 공유**: 외부와 공유가 필요한 파일은 `multica` CLI의 첨부(attachment) 기능이나 댓글 기능을 활용하여 플랫폼 내에 기록하십시오.

**이 규칙을 위반할 경우 시스템 전체의 데이터 일관성이 깨질 수 있으므로 반드시 엄수하십시오.**

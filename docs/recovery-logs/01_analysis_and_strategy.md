# [보고서] 공식 소스 병합 후 기능 장애 분석 및 복구 전략

**일시:** 2026-04-14
**작성자:** Gemini CLI
**상태:** 분석 완료 / 복구 준비 중

---

## 1. 개요 (Overview)
공식 소스(`upstream/main`)의 약 400개 이상 파일(15,000줄 이상)을 기존 워크스페이스에 병합(Merge)한 후, 기존에 잘 작동하던 위키(Wiki) 및 커스텀 에디터 기능이 다수 작동하지 않는 장애가 발생함.

## 2. 장애 원인 분석 (Cause Analysis)

### 2.1 에디터 핵심 로직 변질
- **CSS 스타일 충돌 (`content-editor.css`):** 백업본 대비 약 843줄의 차이가 발생함. 공식 소스의 스타일이 사용자 커스텀 스타일을 덮어쓰면서 레이아웃과 가시성이 완전히 깨짐.
- **실행 로직 간섭 (`content-editor.tsx`):** 약 222줄의 코드 변화. 테이블 관리, 우클릭 메뉴, 협업 커서 등 사용자 고유 로직이 공식 소스의 '버블 메뉴' 및 '링크 프리뷰' 코드와 섞이면서 예기치 않은 동작 발생.

### 2.2 API 클라이언트 규격 변화
- **`client.ts` 및 `ws-client.ts`:** 에러 처리 방식(`ApiError`) 및 인증 로직이 공식 소스 기준으로 바뀌면서, 위키 데이터 저장/조회 시 권한 및 네트워크 통신 오류가 발생할 가능성 확인.

### 2.3 확장 기능(Extensions) 간섭
- 공식 소스의 신규 확장 기능들이 기존 위키 전용 확장 기능들과 충돌하여 에디터 초기화 실패 또는 런타임 토큰 에러(`Invalid token`) 유발.

## 3. 공식 소스 신규 기능 반영 현황 (Added Features)
이번 병합을 통해 다음의 유용한 신규 기능들이 성공적으로 도입되었음을 확인하였으며, 복구 과정에서 이들은 **반드시 보존**되어야 함.
- **Gemini 에이전트 지원:** `server/pkg/agent/gemini.go`
- **OpenCode / OpenClaw 엔진:** `server/internal/daemon/usage/opencode.go` 등
- **개선된 CLI 설치 스크립트:** `scripts/install.sh` 등
- **데스크톱 자동 업데이트:** `apps/desktop/src/main/updater.ts` 등

## 4. 복구 및 대응 전략 (Updated Strategy)

### [전략 B] 타겟팅 정밀 복구 (Targeted Restore) - **[확정]**
백엔드의 신규 기능(Gemini 등)은 그대로 유지하고, 장애가 발생한 프론트엔드 에디터 계층만 백업본으로 복구함.

#### 단계별 실행 계획:
1.  **1단계 (에디터 UI 복구):** `packages/views/editor/` 내의 핵심 로직과 CSS를 백업본(`backup/stable-wiki`) 상태로 강제 복구하여 위키 기능을 정상화함.
2.  **2단계 (백엔드 정합성 유지):** 신규 추가된 Gemini, OpenCode 관련 서버 로직이 UI 복구로 인해 영향받지 않도록 체크함.
3.  **3단계 (정합성 확인):** 위키 기능 및 신규 에이전트 기능 정상 작동 여부 전수 검사.
4.  **4단계 (기능 재이식):** 공식 소스의 유용한 기능(버블 메뉴 등)을 사용자 로직을 해치지 않는 선에서 수동으로 하나씩 재이식.
